The `@keelson/rib-chamber` package is the generative half of
[Chamber](https://github.com/ianphil/chamber) as a
[Keelson](https://danielscholl.github.io/keelson/) **rib**: a discovery-based
extension that brings a multi-agent operating layer to the harness. It contributes
**genesis** (author a persistent agent on demand), **rooms** (orchestrate
agent-to-agent turns), and **agent-authored lenses** (agents that render their own
views through the canvas). The harness stays domain-free; all of the multi-agent
machinery lives here, and the rib ships zero React into the trusted SPA.

These docs mirror the keelson documentation tiers. Keelson documents the generic,
reusable rib contract; this site documents the concrete Chamber rib that implements
it. Where the OSDU rib is the external-system bridge, Chamber is its opposite: a
generative multi-agent surface that proves an agent can author its own lens.

:::note
Documentation is landing tier by tier. The **Concepts** tier is live; the other
tiers fill in from here.
:::
