The reference tier is the exact contract for the Chamber rib: the surface it
contributes, the snapshot keys behind it, the workflows and tools that drive them,
the room strategies, and what the rib persists on disk. It is terse by design; the
[concepts](../concepts/) explain why and the [guides](../guides/) show how.

## Snapshot keys

Every Chamber panel is a board published to a `rib:chamber:*` snapshot key. Eight
keys are standing; three are per-instance families bound at runtime.

| Key | Producer | When it updates |
|---|---|---|
| `rib:chamber:roster` | `chamber-roster` collector | on a 120000 ms cadence, and when a Mind is authored or retired |
| `rib:chamber:rooms` | `chamber-rooms` collector | on cadence, and when a room ends, stops, or is deleted (a starting room appears on the next cadence) |
| `rib:chamber:lenses` | `chamber-lenses` collector | on cadence, and when a lens is authored or retired |
| `rib:chamber:exhibits` | `chamber-exhibits` collector | on cadence, and when an exhibit is tabled or deleted |
| `rib:chamber:brief` | the rib's attention gate | only when a room ends, a lens changes, or an exhibit is tabled |
| `rib:chamber:convene` | the rib (in-process board) | on roster, draft, and convene mutations |
| `rib:chamber:digest` | `chamber-digest` gate + conditional agent turn | on a 120000 ms cadence; spends an agent turn only when the Chamber's structural fingerprint has advanced |
| `rib:chamber:lens-html` | `chamber_emit_lens_html` | on demand, when a Mind emits an HTML canvas board (sandboxed iframe, in-memory, not per-subject) |
| `rib:chamber:room:{slug}` | the room driver | live, as turns land in each active room |
| `rib:chamber:lens:{id}` | `chamber_emit_lens` / `chamber_table_exhibit` | when a Mind authors or re-authors that subject, or a room tables it as an exhibit |
| `rib:chamber:room-view:{slug}` | the `room-open` action | on demand, when a closed room is opened in a drawer |
